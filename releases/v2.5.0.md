# v2.5.0 Release Notes Draft

## Summary

`studyctl v2.5.0` makes the source install and study-session workflow materially more robust, adds first-class Codex CLI support, hardens session database migrations and cross-machine sync behavior, and refreshes the project documentation with current architecture and setup guidance.

## Highlights

- Added first-class Codex CLI support for `studyctl study`, including project-root `AGENTS.md` integration and tmux session support.
- Reworked source installs around typed commands such as `studyctl install tools`, `studyctl install agents`, and `studyctl doctor --fix`.
- Hardened study-session startup and rollback behavior so partial failures clean up tmux, IPC, and DB state correctly.
- Fixed ambiguous write targeting in `session-query tag` and `session-query note` by requiring exact or unique-prefix session matches.
- Improved session DB migration safety with idempotent parked-topic migrations for machines upgrading from older schemas.
- Fixed review scheduling correctness in `review_db`, including missing DB bootstrap and cross-course isolation.
- Improved sync and export correctness for durable study/session metadata and Claude incremental export fingerprints.
- Refreshed user docs, architecture docs, C4 diagrams, and repo code map to match the current package layout and supported agent environments.

## Upgrade Notes

- Existing source installs should refresh with:

```bash
uv sync
uv run studyctl install agents
uv run studyctl doctor --fix
```

- Global CLI installs can be refreshed with:

```bash
./scripts/install.sh
```

- The publish pipeline now cleans `dist/` before building release artifacts.

## Recommended Callouts

- Homebrew remains the preferred end-user install path once the release is published.
- `studyctl` is now versioned at `2.5.0`; the workspace root package version remains separate and is not what the publish workflow ships.
