# Codebase Review Remediation & v2.4.0 Release

**Date:** 2026-04-08 | **Release:** v2.4.0 | **Commits:** `0cca67f`..`f579a69` (18 commits)

## Problem Statement

A comprehensive codebase review (5 parallel analysis agents covering architecture, security, tests, packaging, and code quality) found **23 issues**:

| Category | Issues | Examples |
|---|---|---|
| Architecture | 7 | Layer violations (session/web importing from cli/), mixed UTC/local timestamps |
| Security | 3 | Path traversal in `list_artefacts`, no SAST, unpinned CI action tags |
| Tests | 3 | Missing `e2e` marker, 1,162 LOC ML code with zero tests, dead test files |
| Packaging | 2 | Broken `install.sh` version check (3.10→3.12), broken `install-agents.sh` |
| Code Quality | 5 | 4 complexity hotspots (CC 19-21), dead `web/server.py` (259 LOC), module-level side effects |
| Exporters/Data | 3 | Archive schema drift (5 missing columns), protocol inconsistencies, existence-only re-import |

One false positive retracted: FTS5 `bm25()` returns negative values — `ASC` sort is correct.

## Root Causes

**Layer violations:** `session/resume.py`, `web/routes/session.py`, and `session/orchestrator.py` imported from `cli/` — lower-level modules depending on higher-level ones. Fixed via callback injection and moving shared logic to `session/cleanup.py`.

**Complexity hotspots:** Four functions (CC 19-21) grew incrementally without a complexity budget. Each mixed 3-5 concerns in a single function body.

**Missing ML test coverage:** `embeddings.py` (593 LOC) and `semantic_search.py` (569 LOC) grew incrementally with tests deferred each time. Cost of retrofitting: one full session.

**Dead code accumulation:** Features toggled off or superseded without removing the old path. `web/server.py` (stdlib HTTP, superseded by FastAPI) had zero callers for months.

**Missing Alpine.js stores:** A layout redesign rewrote the HTML but `$store.pomodoro` and `$store.settings` were never recreated — broke the entire web UI on load.

## Key Technical Details

### Adapter Plugin Architecture

`studyctl/adapters/` — Protocol-based (structural typing, no ABC inheritance):
- 6 built-in adapters as individual files with auto-discovery via `pkgutil.iter_modules`
- Config-driven custom agents (`agents.custom` in config.yaml)
- 3 persona strategies: `cli-flag`, `cwd-file`, `mcp-config-writer`
- `agent_launcher.py` retained as backwards-compatible facade (585→396 LOC)

### Refactoring Techniques

- **Data-driven mapping:** Replace per-field if-blocks with `dict[str, tuple[str, Callable]]` iterated once
- **Function extraction + typed exception:** `SessionStartError` replaces scattered `ctx.exit(1)` calls
- **Decomposition:** Single-responsibility helpers with orchestrator as 14-line coordinator
- **Module splitting:** Natural seams — DB access vs business logic vs continuation-specific logic

### CI Hardening

- **bandit SAST:** Pre-commit + CI, skipping noise categories (subprocess, SQLite, test fixtures)
- **SHA pinning:** `actions/checkout@11bd7190` (v4.2.2), `astral-sh/setup-uv@e4db8464` (v4)
- **Automated Homebrew:** New `update-formula` job in `publish.yml` — downloads tarball, computes SHA, commits

## Prevention Strategies

### Automated Guards Now in Place

- **bandit SAST** in pre-commit + CI catches security anti-patterns
- **SHA-pinned actions** prevent supply chain attacks via tag manipulation
- **`uv audit`** in CI catches vulnerable dependencies before merge
- **pyright on `src/`** in both pre-commit and CI gates type regressions
- **Adapter Protocol** enforces interface contracts at the type level

### Architectural Patterns Established

- **Protocol-based adapters** — define interface in Protocol, implement in adapters/, inject at CLI/web layer
- **Lazy initialisation** — `_connect()` / `_get_model()` helpers, never at import time
- **Callback injection** — invert dependencies without extracting monolithic functions
- **Complexity budget** — CC >15 must be split before merge

### Remaining Risks

- **ML golden-dataset tests** — current tests mock the model; no regression test against actual similarity rankings
- **Complexity re-accumulation** — `cli/_study.py` and SM-2 scheduler are the most likely next hotspots
- **Dead code** — quarterly `vulture` run recommended to catch accumulation
- **Alpine.js store inventory** — no canonical registry of which stores exist and which components depend on them
- **Legacy timestamp data** — existing rows have mixed UTC/local; one-time migration not yet written

### Process Learning

**Parallel agent dispatch** is the default for multi-issue remediation. 7 agents on independent files cut ~90min sequential work to ~25min wall-clock. Pattern: triage into independent tracks → scoped brief per track → parallel dispatch → single integration gate.
