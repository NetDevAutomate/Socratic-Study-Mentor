# Generated Artefacts

AI-generated overviews of this project via [Google NotebookLM](https://notebooklm.google.com).

These are auto-generated using [notebooklm-repo-artefacts](https://github.com/NetDevAutomate/notebooklm-repo-artefacts) and hosted on the [artefact store](https://artefacts.netdevautomate.dev/socratic-study-mentor/artefacts/).

## Available Formats

| Format | Description |
|--------|-------------|
| [Audio Overview](https://artefacts.netdevautomate.dev/socratic-study-mentor/artefacts/) | Two AI hosts discuss the project — great for commutes |
| [Video Overview](https://artefacts.netdevautomate.dev/socratic-study-mentor/artefacts/#video) | Visual walkthrough of architecture and concepts |
| [Infographic](https://artefacts.netdevautomate.dev/socratic-study-mentor/artefacts/#infographic) | Architecture and flow at a glance |
| [Slide Deck](https://artefacts.netdevautomate.dev/socratic-study-mentor/artefacts/#slides) | Presentation-ready project overview |

## How Artefacts Are Generated

The `studyctl content` pipeline and the `notebooklm-repo-artefacts` tool work together:

1. **Collect** — Repository content is gathered and formatted
2. **Upload** — Source material is uploaded to a NotebookLM notebook
3. **Generate** — NotebookLM creates audio, video, slides, and infographics
4. **Download** — Generated artefacts are downloaded locally
5. **Publish** — Artefacts are pushed to the [artefact store](https://artefacts.netdevautomate.dev)

See the [content pipeline documentation](content-pipeline.md) for more details on the `studyctl content` commands.
